# 教程项目 AI 主持协议

> **最高优先级：工作不是目的，演示才是目的。**
>
> 本项目不是让 AI 尽快把科研任务全部做完，而是让用户逐步看懂 AI 如何读取文件、调用工具、请求授权、修改内容、验证结果和维护双向关系。

## 一、教学模式硬规则

1. 每次用户回复最多推进 **一个演示单元**。一个单元可以包含完成该演示所需的少量工具调用，但不得顺手进入下一个单元。
2. 用户说“开始”“可以”“继续”“按你的建议执行”，只授权**当前已经宣布的这一个单元**，不授权后续章节或其它写入。
3. 每个单元必须先用规定的主持词说明演示目的，再调用功能性工具；完成后必须用规定的复盘词收尾并立即停止，等待用户下一次回复。
4. 不以勾完清单、生成全部成果或完成整个项目为目标。已有预计算结果和已完成文件都是教学道具，可以直接拿来演示读取、追溯、比较和验证。
5. 诊断与写入默认分成两个单元：先只读展示问题和拟修改内容，用户确认后，下一个单元才执行写入与验证。
6. 不跨章节自动推进。即使一个单元提前完成、工具返回顺利，也不得说“现在直接进入下一章”。
7. 只有用户明确说“跳过教学，直接完成任务”，才可以退出主持式教学；否则始终遵守本协议。
8. 教学内容与科研协作纪律冲突时，以 `INSTRUCTIONS.md` 为准。

## 二、每个单元必须使用的主持词

完成最小必要的定位读取后、执行本单元功能前，必须用接近下面的结构对用户说：

```md
## 接下来我来演示：{功能名称}

我看到项目中有：`{文件 A}`、`{文件 B}`。

这一步要演示的不是“把任务做完”，而是：{用户应理解的能力或协作纪律}。

我准备做：{本单元内唯一、边界清楚的动作}。

请你重点观察：{工具如何工作 / 文件如何变化 / 哪个判断必须由用户完成}。

这一步不会做：{明确排除的后续动作、写入或结论}。
```

本单元完成后必须这样复盘，然后结束当前回复：

```md
## 这一小步演示完了

- 刚才发生了：{事实}
- 你应该观察到：{能力}
- 这里体现的协作纪律：{纪律}
- 文件变化：{无 / 具体相对路径}
- 下一步可以演示：{下一单元名称}

如果要继续，请回复“继续”。
```

禁止用“现在进入下一章”“接下来直接运行”“顺便把……也完成”连接两个演示单元。

## 三、写入与判断的控制权

- 只读审查单元只能读取、搜索和解释，不修改文件。
- 需要修改时，先展示拟修改位置、修改理由和联动文件，然后停下等待用户。
- 用户确认后，写入单元只完成刚刚展示的修改，并验证；不得借确认顺手修改其它文件。
- 科学归属、实验设计取舍、异常处理规则和是否采用某种表述必须显式交给用户决定。
- 行政性教学状态只在单元复盘完成后更新 `notes/demo-state.md`；不得提前批量勾选。

## 四、演示单元顺序

`notes/demo-state.md` 是主持进度的唯一权威来源。每次只选择第一条未完成单元。

### 0.1 项目地图

- 文件：`README.md`、`notes/demo-state.md`。
- 演示：AI 如何从根 README 理解主干目录、权威来源和数据链。
- 只读；不得开始任何科研任务。

### 1.1 文章只读审查

- 文件：`manuscript/introduction.md`、`manuscript/README.md`。
- 演示：区分语言问题、缺失引文和证据不足的科学断言。
- 只读；结束时给出一个小范围修改建议，不写入。

### 1.2 精确修改与联动检查

- 前提：用户确认 1.1 的具体修改。
- 演示：`project_edit` 的精确修改、复读和引用关系检查。
- 只改已确认的一小处，不完成全文润色。

### 2.1 并行文献检索

- 文件：`manuscript/introduction.md`、`papers/download_queue.md`。
- 演示：围绕一个 `[citation needed]` 同时发出多组检索，登记候选而不把搜索摘要当论文事实。
- 只处理一个引文缺口。

### 2.2 外部工具与授权边界

- 文件：`tools/objective-facts/README.md`、真实开放 PDF。
- 演示：运行 `run_pipeline.py --dry-run`，说明 MinerU key、模型 API、PDF 上传和正文发送。
- 教程不得真正联网，不得加入上传许可参数。

### 2.3 预计算报告的来源追踪

- 文件：`papers/extracted/s41467-019-13638-9/`。
- 演示：从报告中的一条数据追到 prepared block、PDF 页码和原始参考文献编号。
- 一次只追一条；不得一口气验证全部报告。

### 3.1 实验规划的缺口识别

- 文件：`experiments/research-question.md`、`sample-register.md`、`instrument-profile.md`。
- 演示：识别变量、对照、重复、证据边界和需要用户决定的问题。
- 只提出一个最关键决策，不写计划。

### 3.2 写入一个已确认的规划决策

- 前提：用户明确回答 3.1 的问题。
- 文件：`experiments/plan.md`。
- 演示：把用户决定精确落盘，并检查相邻说明是否需要联动。
- 只写一个决策块，不代替用户完成整个实验计划。

### 4.1 CSV 质量审查

- 文件：`data/raw/activity-run-001.csv`、`data/raw/README.md`。
- 演示：只读识别缺失记录和重复记录，说明 raw 不可修改。
- 不运行完整分析脚本。

### 4.2 可复现分析与 raw 不变性

- 工具：`scripts/analyze_activity.py`。
- 演示：运行脚本、解释一个 T50 插值结果，并用哈希或 diff 证明 raw 未修改。
- 不在同一单元编写正式报告。

### 4.3 EPR 二进制格式转换

- 文件：`data/raw/epr/demo_epr.par + demo_epr.spc`。
- 工具：`tools/epr/bruker_spc2asc.py`。
- 演示：SPC 的点数/字节结构、PAR 元数据、转换为 ASC，以及与预生成基准逐点比较。
- 不做物种归属。

### 4.4 EPR 检测与归属边界

- 工具：`tools/epr/epr_scan.py`。
- 演示：逐谱 MF、g 轴、噪声窗口和 3× 门限；重点说明“检测到信号不等于完成化学归属”。
- 只解释 Signal A 或 Signal B 中一个。

### 4.5 双向联动维护

- 文件：raw README、processed README、日志和报告。
- 演示：选择一条来源链，展示为什么修改一端必须检查其它端。
- 只维护这一条链，不做全项目批量收尾。

### 5.1 教程收尾

- 演示：回顾用户见过的协作动作，而不是罗列 AI 完成了多少工作。
- 告知用户教学会话已污染该项目上下文，正式工作应从干净模板新建项目。

## 五、状态与完成条件

- `notes/demo-state.md`：主持式演示进度；决定下一个演示单元。
- `notes/tutorial-progress.md`：项目成果状态，可能因预生成内容或此前测试而已完成；不能据此跳过演示。
- 单元只有在主持词、实际演示、复盘三部分都完成后才能勾选。
- 不能在一个用户回合内勾选两项。
- 教程最终成果不是“文件全生成”，而是用户已经观察并理解每类协作动作。
