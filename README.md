# Workmode Public

> 让 AI 在真实科研项目里工作，而不只是停留在聊天框里。

Workmode Public 是一个可独立安装、本地优先的科研工作台。它把项目文件、论文阅读、AI 工具执行、会话历史和长期工作上下文放在同一个桌面应用中，面向真实科研工作，而非人格陪伴或生活记忆。

当前正式支持 **Windows x64**。安装版已包含运行环境，普通用户不需要预装 Node.js、Python 或 Rust。

[下载最新版本](https://github.com/carbocation123/workmode-public/releases/latest) · [查看更新记录](https://github.com/carbocation123/workmode-public/releases) · [开发文档](docs/DEVELOPMENT.md) · [反馈问题](https://github.com/carbocation123/workmode-public/issues) · [安全策略](SECURITY.md)

## 为什么是 Workmode Public

- **围绕真实项目工作**：AI 可以在用户授权的项目目录中读取、搜索、创建和精确编辑文件，而不是把所有材料复制进一个封闭知识库。
- **科研过程可追踪**：会话、工具调用、停止状态、上下文压缩和项目记忆都有明确记录；压缩上下文不会删除原始历史。
- **本地数据由用户控制**：项目、论文、会话、配置和诊断报告保存在本机；只有用户主动使用的模型、MinerU、网页或更新功能会产生对应网络请求。
- **开箱即用的桌面体验**：提供 Windows 安装包、首次使用向导、应用内更新和一键错误报告，不要求用户自行配置开发环境。

## 四个功能入口

| 模式 | 适合谁 | 主要能力 |
| --- | --- | --- |
| **科研工作台** | 需要完整项目控制和可组合 AI 工具的用户 | 项目与会话、文件树、Markdown/PDF/图片预览、受控文件编辑、Shell/Python、网页检索、项目记忆、上下文查看与压缩 |
| **文献智库** | 希望快速建立本地论文库并围绕文献对话的用户 | PDF 入库与去重、论文选择、原文阅读、单篇精读默认逐图讲解、标签与笔记、多会话、回收站、可选 MinerU 增强解析和跨文献协作 |
| **会议录音转文字** | 只想批量上传录音并集中查看结果的用户 | DashScope Fun-ASR、说话人区分、多文件队列、失败重试、TXT/Markdown/JSON 导出、手动 AI 润色/总结和可恢复删除 |
| **文章处理** | 需要快速润色文字或检查长文内部问题的用户 | 文字润色、证据链/逻辑/表述一致性核查、Unicode 上下标、本地处理历史和可恢复删除 |

科研工作台与文献智库共享同一套项目注册、会话历史、模型配置和上下文机制。会议转写是无 session 的独立文件工具，只读取固定 `input/` 与 `output/`；即使同一目录后来由通用工作台打开并增加其它文件，也不会改变转写列表或任务状态。文章处理同样不创建项目或对话，成功结果作为独立的本地历史记录保存。

## 核心能力

### 项目化 AI 工作

- 注册和切换多个本地项目，并按真实父子目录关系展示项目。
- 每个项目拥有独立会话、项目工作记忆、任务计划和上下文状态。
- 模型可调用受控工具读取、搜索、创建和编辑项目文件，也可在项目目录运行带超时、取消和输出限制的 Shell/Python。
- 文件查看器支持 UTF-8 文本、Markdown、PDF 和常见图片；编辑已有 Markdown 时使用内容版本校验，避免覆盖外部修改。

### 长期上下文与可追踪历史

- 使用 `@相对路径.md` 将项目协议、实验约束或其它 UTF-8 文本固定注入上下文。
- 按 token 预算动态选择最近历史，并在界面显示历史消息、固定文件和摘要占用。
- 手动压缩上下文时生成明确的摘要续接点，完整 JSONL 历史仍然保留。
- AI 回复、工具开始、工具结果和中途停止产生的内容按真实到达顺序持久化。

### 本地文献工作流

1. 只填写名称即可创建文献项目。Windows 有 D 盘时默认保存在 `D:\workmode\<项目名>`，否则使用 `~/workmode`；旧版外部路径项目仍可继续使用。
2. 可以选择 `.enl` 或 `.enlx` 把 EndNote 文献库合并进当前项目。自动查找会扫描本机全部固定盘和可移动盘，不依赖任何网盘或用户目录硬编码；同目录同名的 `.enl`、`.Data` 和 `.enlx` 聚合显示并优先完整工作库。导入前会预检并提醒关闭 EndNote；标题、作者、年份、发表日期、期刊、DOI、手工分组、真实彩色标签和附件在本机直接迁移，不调用模型或 MinerU，也不修改原 EndNote 文件。EndNote 分组集会按“分组集 - 手工分组”拍平，智能分组不会导入。
3. EndNote 每条记录会从全部附件中寻找第一个有效 PDF 作为主论文，其余附件放入该文献的 SI 文件夹；没有有效 PDF 的记录单独失败，不中断其它文献。导入后可按 DOI、主 PDF SHA-256、标题+年份+第一作者一键查重，结果只供人工处理，不自动合并。
4. 拖入 PDF 后先检查文件清单，只有点击“确认入库”才写入项目，并使用 SHA-256 去重。
5. 入库不会自动选择论文、调用模型或启动 MinerU。用户可以选择论文或笔记作为当前对话资料，再自然语言提问。
6. 文献列表、详情页和 AI 工具读取同一份字段契约。标题、作者、第一作者姓、年份/发表日期、期刊/缩写、DOI、文章类型、文献分组、彩色标签、用户关注点、AI 摘要、主 PDF、SI、解析产物和处理/校验状态会保持同一语义；缺少作者、年份或期刊的导入记录会如实标为元数据不完整。AI 查询文献库结构时会明确区分 `groups.json` 的文献分组与 `tags.json` 的标签组，不会再根据物理文件夹猜分组。
7. 左侧列表使用统一的搜索与操作栏：分组、标签和导入保持同一层级，PDF 与 EndNote 入口收在同一个导入菜单中。卡片默认只显示标题、期刊、年份和少量分组/标签，不再重复显示原始文件名；完整字段仍可在详情中查看。
8. 选中一篇论文后说“精读这篇”，AI 会先读取真实正文和图注，再默认按图 1、图 2……讲解每幅图及各 panel 做了什么、观察到什么、如何支撑结论。图表信息不足时会明确说明，不会猜测。
9. 普通阅读优先使用已有解析文本，否则直接读取 PDF 文本层；需要更可靠地恢复复杂图表或版面时，AI 会提示是否使用 MinerU，不会未经同意自动启动耗时解析。
10. 论文记录、主 PDF、SI 和解析产物可以整体移入项目回收站并无覆盖恢复；历史会话不会被改写。

### 会议录音转文字

1. 在共享设置页保存 DashScope API Key；密钥不会回显，也不会写入录音目录或转写结果。转写指引中的百炼控制台、API Key 教程和免费额度按钮会通过系统浏览器打开阿里云官方页面；若系统浏览器调用失败，页面会显示可手动复制的完整地址。
2. 一次选择一个或多个 M4A、MP3、WAV、OGG、FLAC、AAC、WebM 或 MP4 文件；原始文件保存在 `input/<任务 ID>/`。
3. 后端以单工作线程调用 Fun-ASR 并区分说话人，任务编号写入 `output/<任务 ID>/meta.json`，应用重启后可以继续未完成任务。
4. 前端集中显示带时间戳的说话人分段、纯文本和 Markdown，并可下载 TXT、Markdown、JSON 或直接定位输出文件夹。
5. 转写完成后可以手动选择“AI 润色”或“AI 总结”。调用前会明确提示文本将发送给用户配置的模型服务且可能产生费用；长转写会分段处理，结果分别保存为 `ai-polished.md` 与 `ai-summary.md`，不会覆盖原始 ASR 文件。
6. AI 结果可查看、下载、重新生成或单独清除；总结固定区分核心结论、决定事项、行动项与待确认问题，重要内容仍应人工核对。重新转写成功后旧 AI 派生结果会自动失效，避免错配。
7. 标题可以修改；删除会把对应输入与输出整体移入 `output/.trash/`，可从页面恢复。模块不会创建项目、对话或 JSONL。
8. 第一次进入时会播放四步使用指引：DashScope 新手申请、批量上传、结果导出，以及可选 AI 润色/总结。指引写明账号与 Key 申请、地域、费用保护、模型连接测试、常见错误、隐私边界和成功标志；右上角“使用指引”或共享设置页都可以重新播放。引导进度只保存在当前 WebView 的本地存储中，不写入转写工作目录。

### 文章处理

1. 在功能大厅打开“文章处理”，选择“文字润色”或“查找漏洞”，粘贴文字后手动开始处理；页面左上角始终可以返回功能大厅，包括启用 HUD 或官方签名皮肤时。
2. 文字润色会改善学术表达，但要求保留原文的事实、数字、引文、结论强度与段落结构；查找漏洞会检查文章内部的证据链、逻辑跳跃、概念漂移和前后表述一致性。
3. 长文核查会先分段分析，再合并为全篇报告；它不会联网核验外部事实或参考文献真伪，重要问题仍需人工回到原文和来源核对。
4. 明确的上下标写法会尽量转换为可以直接复制进 Word 的 Unicode 字符，例如 `H₂O`、`10⁻³`、`R²`，不使用 `**₂` 一类标记语法。
5. 只有成功完成的请求才写入本地历史。每条历史保留当时的输入、功能、模型和输出，可以重新打开、再次处理、移入“已删除记录”并恢复；“新任务”会保留当前处理模式并清空编辑区，不会删除既有历史；模块不创建 Workmode project、session 或 JSONL。

完整的新手步骤、隐私与费用边界见 [文章处理使用指引](docs/ARTICLE-PROCESSING.md)。

## 三分钟开始

1. 从 [GitHub Releases](https://github.com/carbocation123/workmode-public/releases/latest) 下载 `workmode-public-<version>-windows-x86_64-setup.exe`。
2. 安装并启动 Workmode Public，在功能大厅选择“科研工作台”“文献智库”“会议录音转文字”或“文章处理”。
3. 在首次向导中填写 OpenAI-compatible Base URL、模型名和 API Key；也可以使用内置的 DeepSeek 配置引导。
4. 创建教程项目、打开自己的项目，或新建一个本地文献库。
5. 根据界面引导开始对话、阅读文件、导入论文、转写录音或处理文章。所有向导都可以跳过，也可以稍后在设置中重新播放。

模型工具循环没有固定轮数上限。它会持续到模型给出最终正文、用户点击停止，或发生无法继续的错误；普通工具失败会返回给模型，由模型决定修正参数、改用其它方法或向用户说明。

## 本地优先与数据边界

正式桌面版默认将状态保存在：

```text
%LOCALAPPDATA%\WorkmodePublic\
  config\.env          模型 API 与本地配置
  data\                项目注册、会话、记忆、计划、文章处理历史和历史备份
  logs\runs\<run-id>  最近 20 次桌面运行日志
  reports\             用户主动生成的错误报告 ZIP
```

| 数据或行为 | 默认边界 |
| --- | --- |
| 项目文件、PDF、会话、记忆和计划 | 保存在本机，不由应用自动上传 |
| 模型对话、转写 AI 处理与文章处理 | 仅在用户发送消息、确认处理已完成转写，或点击文章“开始处理”时请求用户配置的模型服务；请求会包含完成当前操作所需的上下文、转写文本或文章文本 |
| MinerU | 可选；仅在用户明确要求增强解析时使用配置的 MinerU 服务 |
| DashScope Fun-ASR | 仅在用户上传录音并提交转写时使用；音频会上传到用户配置的 DashScope 服务，签名文件地址不会写入本地任务文件或日志 |
| 网页检索 | 仅在模型调用网页工具时访问公开 HTTP(S) 页面 |
| 运行日志与错误报告 | 保存在本机；报告 ZIP 由用户主动生成和发送，应用不会自动上传或删除 |
| 应用更新 | 用户检查更新时访问 GitHub Releases，下载内容经过 Tauri 更新签名验证 |

默认后端只绑定 loopback。PDF 和图片预览使用扩展名、大小与 magic bytes 白名单；网页工具拒绝 loopback、内网、链路本地地址、非常用端口和非 HTTP(S) 协议，并在重定向后重新验证目标。

设置页的“一键生成错误报告”只收集本次桌面运行日志，限制单项日志尾部大小，并再次脱敏 Token、API Key、密码和 Windows 本地路径。生成后，Windows 文件管理器会定位 ZIP，是否发送以及何时删除都由用户决定。

## 从源码运行

最简单的浏览器源码入口：

```powershell
git clone https://github.com/carbocation123/workmode-public.git
cd workmode-public
.\start-workmode-public.cmd
```

停止源码后端：

```powershell
.\stop-workmode-public.cmd
```

源码启动器会按需创建 Python 虚拟环境、安装依赖、构建前端、启动本地后端并打开浏览器。Tauri 桌面开发、测试矩阵和正式发行步骤见 [开发环境与测试](docs/DEVELOPMENT.md) 与 [桌面发行与自动更新](docs/DESKTOP-DISTRIBUTION.md)。

## 当前边界

- 正式安装包目前只发布 Windows x64。
- 项目 Shell/Python 工具有工作目录、超时、输出截断、取消和危险命令限制，但不是操作系统级沙箱。
- 文献智库仍缺真实 MinerU 联网验收、DOI 级语义去重、候选标签治理和研究论文/综述最终主档生成。
- 会议转写当前处理已有录音文件，不提供实时录音或逐字稿在线校对；AI 润色与总结是手动触发的模型派生结果，不代替原始转写或人工确认。说话人编号是单次录音内的匿名编号。
- 文章处理只核查输入文本内部的表达、证据链、逻辑和一致性，不联网检索，也不能证明外部事实、实验数据或参考文献真实可靠；Unicode 上下标转换只覆盖含义明确的常见写法。
- 尚未提供子 agent；需要先明确权限、并发、预算、取消和文件冲突语义。
- Tauri 更新内容经过签名验证，但 Windows 安装器尚未配置 Authenticode，首次下载可能触发 SmartScreen 提示。
