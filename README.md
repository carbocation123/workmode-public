# Workmode Public

Workmode Public 是一个可独立安装、本地优先的纯净科研工作助手。它围绕项目文件、科研写作、实验记录、数据处理和可追踪的长期工作上下文设计，不包含私人伴侣、生活记忆或人格关系模块。

[下载最新 Windows 版本](https://github.com/carbocation123/workmode-public/releases/latest) · [查看发行记录](https://github.com/carbocation123/workmode-public/releases) · [报告安全问题](SECURITY.md)

## 0.8.4 更新摘要

- 设置页重排为尺寸、间距和圆角统一的两列卡片布局，问题反馈并入「桌面应用与支持」，窄屏自动切换为单列；
- 文献智库改为轻量默认流程：导入只登记 PDF，不自动选择或启动 MinerU；普通提问按需读取原文，增强解析、事实抽取和归档只在用户明确要求时运行；
- 文献项目可直接填写名称创建到应用托管目录，同时保留旧版任意路径项目；新增论文回收站、无覆盖恢复、会话重命名和选择历史事件；
- `literature_read` 支持一次读取最多 20 篇论文，`literature_process` 支持最多 3 路并发处理；元数据 JSON 解析、证据核验和失败隔离更稳健；
- 修复 Windows 安装版 PDF 拖放，并优化功能大厅、文献时间线自动跟随、工具写入后的即时刷新以及 DeepSeek/MinerU 配置引导；
- 从 0.7.1 起可直接在应用内更新，无需迁移项目、会话或设置。奖励皮肤仍由维护者单独发放，不进入 GitHub Release。

## 0.8.4 详细改进

- 功能大厅移除说明面板内容；两张入口卡以受控高度在顶栏与底栏之间垂直居中，底栏仍固定在第四行，不会因删文案而被纵向拉长或与旧皮肤错位；科研工作台使用“定制你自己的AI工作流”，文献入口明确标记为“文献特化模块”；
- 文献智库对话顶栏移除“和文献一起讨论”这类冗余拟人化副标题，只保留模块标识与实际会话、上下文和笔记控件；
- 设置页改为平衡的两列卡片布局：模型与 MinerU 并排，外观和项目记忆横跨整行，支持、引导和连接按固定区域排列；卡片宽度、内边距和圆角统一，问题反馈合并进“桌面应用与支持”，不再单独占用一个网格；窄屏时按相同信息层级切换为单列；
- DeepSeek 与 MinerU 引导补充了官方页面中的具体按钮、Key/Token 创建和粘贴位置。MinerU 仍是可选增强，没有 Token 时继续使用 PDF 文本层；
- 文献时间线采用类似 VS Code 的“跟随最新”逻辑：用户停留在底部时自动跟随，主动向上滚动后停止抢滚动，并显示「回到最新」；
- 文献领域工具成功写入后依据 `changed_paths` 即时刷新论文、标签或笔记投影，不必等待整轮对话结束或手工刷新；
- 多篇论文可由一次 `literature_process(paper_ids=[...])` 以最多 3 路并发处理。每篇结果相互隔离，一篇失败不会取消其余论文；
- 元数据抽取请求模型返回 JSON object，解析失败时只进行一次受控 JSON 修复，并保存原始/修复响应用于排错。首页元数据无法逐字核验时，论文转入“元数据待人工确认”，但 MinerU 产物与客观事实报告继续生成。
- 文献智库的默认路径改为轻量对话：每个新 session 以一条正式 AI 自我介绍开场；确认入库只登记 PDF，不自动勾选、不启动 MinerU 或事实抽取；导入和文献选择会在下一次用户消息前写成永久 system 事件并显示在时间线中；普通问题按需调用 `literature_read`，默认以“实验现象 / 研究手段 / 关键证据 / 说明的问题”简洁回答；
- `literature_read` 现在同时接受一个 `paper_id` 或最多 20 个 `paper_ids`，多篇概括或比较可用一次只读工具调用完成。MinerU、标准命名、标签、事实报告与归档仍保留为用户明确要求时才进入的高级流程；默认文献卡不再显示“待处理 / 未处理 / 标准命名待确认”等入档状态占位。
- `literature_update_record` 接受常见的人类期刊缩写写法：点号、空格和标点会在生成标准文件名时自动移除，例如 `Angew. Chem. Int. Ed.` 会规范为 `AngewChemIntEd`；校验失败发生在 catalog/tags 写入之前，不再出现“工具显示失败但错误参数已部分落盘”的情况。
- 文献详情现在可以把论文移入项目回收站；`catalog.json` 记录、原始 PDF 与 MinerU/事实抽取目录作为一个事务一起移动到 `papers/.trash/`。文献库顶部可查看并恢复，恢复绝不覆盖已有文件；当前界面选择会立即清理，历史 session/JSONL 保持原样。

## 主要能力

- 注册并切换多个本地项目，按真实父子目录关系展示项目；
- 为每个项目保存独立会话、项目工作记忆和任务计划；
- 浏览默认折叠的项目文件树，以图标区分目录、Markdown、PDF、图片、代码、实验数据和压缩包；预览 UTF-8 文本、Markdown、PDF 和常见图片；
- 在 AI 回复和文件预览中渲染 GitHub Flavored Markdown 表格与列表，宽表可横向滚动，列表符号保持在气泡内容区内；编辑已有 Markdown 时使用内容版本校验避免覆盖外部修改；
- 让模型读取、搜索、创建和精确编辑项目文件，也可以在项目目录运行受限的 shell 与 Python；
- 使用 `web_search` 通过免 Key的 DuckDuckGo HTML 端点并行检索公开网页，再用 `web_fetch` 核对原始页面；检索可用性取决于用户当前网络环境；
- 在工作记忆中用 `@相对路径.md` 固定注入项目协议或其他 UTF-8 文本；
- 按 token 预算动态选择最近历史，并在界面显示估算总量、历史数量、固定导入文件数和摘要状态；
- 手动压缩上下文，保留完整 JSONL 历史，只改变后续模型的续接起点；
- 停止正在运行的生成和工具链，并按原始“文字 → 工具 → 文字”顺序持久化已产生内容；
- 从 GitHub Releases 检查、下载并安装经过 Tauri 更新签名验证的新版本；
- 科研工作台首次进入时逐步配置并测试模型；内置 DeepSeek 开放平台注册、充值、创建 API Key 教程和 V4 Pro/Flash 一键预设；随后选择教程或自己的项目，再通过六步高亮指引认识界面；
- 文献智库具有独立的四步新手引导，说明 PDF 确认入库、默认按需读取、可选 MinerU 增强、资料选择和自然语言协作；MinerU 的具体 Token 配置保留在共享设置中，不再作为普通阅读的前置步骤；两套引导都可在设置中单独重新播放，不会互相串场；
- 从应用内创建官方科研协作教程，使用六项真实任务卡体验完整流程，并在测试后安全重置到内置初始态；
- 启动后先进入功能大厅：重度用户选择「科研工作台」，轻量用户选择「文献智库」；文献页只需填写名称即可建立应用托管的固定结构文献库，也可继续打开旧版外部项目，文献卡片、标签筛选、PDF/事实报告、笔记、多 session、上下文压缩和工具事件都复用正式 Workmode 内核；
- 完整工作台与文献智库使用同一套左侧活动栏，只保留「功能大厅」与「设置」两个全局入口；文献设置复用同一全局设置页，并提供 MinerU Token、解析模型、语言和超时配置，关闭时返回原文献页面；
- 文献卡片、项目笔记和项目记忆都在页面中央打开独立窗口；文献的「原始 PDF」页签与完整工作台复用同一个受控 PDF 阅读器和媒体端点，阅读器先在应用内获取文件再用 `blob:` 地址显示，避免第三方下载软件截获 iframe 导航；
- 用本地成就记录模型连通、项目创建、PDF 阅读、网页检索、脚本分析、Markdown 保存、上下文查看与压缩等真实操作。
- 提供实验室、Origin Ring、Neon Space Lab、论文纸、深夜观测站和高对比内置皮肤，并支持导入多个官方 Ed25519 签名的 `.workmode-skin`；签名皮肤可分别控制真实内容槽位的布局与视觉，切换时无需重新安装应用。Neon Space Lab 的共享舰桥 HUD 与结构材质会同时覆盖功能大厅、完整工作台和文献智库，不再只改变特化页面的配色。
- 官方签名皮肤可以携带布局 CSS、视觉 CSS、字体、图标和图片，但不能携带 JavaScript、HTML 或权限；未知签名、内容被修改和旧式 JSON 皮肤都会被拒绝。
- 在设置页一键生成本次运行的脱敏错误报告 ZIP，并让 Windows 文件管理器直接定位文件；用户可自行拖入微信、邮件或其它反馈渠道，应用不会自动上传报告。

模型工具循环没有固定轮数上限。它会持续到模型给出最终正文、用户点击停止，或发生无法继续的模型/网络错误；普通工具失败会作为结果返回模型，由模型决定修正参数、换方法或向用户说明。

## 安装与首次使用

当前正式分发目标是 Windows x64。普通用户只需要从 [Releases](https://github.com/carbocation123/workmode-public/releases/latest) 下载：

```text
workmode-public-<version>-windows-x86_64-setup.exe
```

目标电脑不需要安装 Node.js、Python 或 Rust。安装完成后：

1. 从开始菜单启动 Workmode Public，先在功能大厅选择使用方式：「科研工作台」提供完整项目和工具能力，「文献智库」提供固定文献工作流；
2. 首次进入文献智库时，独立向导会说明确认入库、按需阅读、可选的 MinerU 增强解析和对话资料选择；每个新文献 session 还会用一条简短自我介绍说明可用能力；
3. 首次进入科研工作台时，启动向导会说明本地数据和模型请求边界；
4. 没有模型 API 时，展开「如何申请 DeepSeek API」，依次登录开放平台、充值 API 余额并创建 API Key；也可以继续填写其它 OpenAI-compatible Base URL、模型名和 API Key；
5. 选择「体验科研教程（推荐）」或「打开自己的项目」，并跟随六步界面高亮认识项目、文件、时间线、上下文、文件查看器和输入框；
6. 教程项目右下角会显示六项真实任务，示例指令只填入输入框，由用户检查后按 `Ctrl+Enter` 发送；后续可在设置页管理模型、MinerU、外观、皮肤、成就、更新和问题反馈。

向导每一步都可以跳过，不会锁住已经熟悉产品的用户。科研工作台与文献智库的引导状态分别保存在本机，不进入项目文件、模型上下文或网络请求。教程任务和成就仍属于科研工作台。通过 Vite 开发服务器、`tauri dev` 或一键源码启动器运行时会自动补齐全部成就，方便维护者检查奖励主题与皮肤；正式 Release 构建和桌面安装包仍只根据真实产品操作解锁。

皮肤选择和降低动效同样只保存在桌面 WebView 本机。实验室、论文纸、深夜观测站、高对比和跟随系统始终可用；完成官方科研协作教程后会解锁 Origin Ring 与结构皮肤 Neon Space Lab。亮色、暗色与高对比等基础可访问性选项不会被成就锁定。Neon Space Lab 的界面与纹理由项目原创 React/CSS/SVG 实现，不分发或执行参考皮肤中的程序、脚本、字体或图片。

Workmode 0.7.0 起采用官方签名皮肤协议。每个 `.workmode-skin` 同时包含 `layout.css` 与 `visual.css`：前者只重新排列应用维护的项目、文件、会话、消息、工具、输入区、查看器和设置等真实语义槽位，后者负责颜色、字体、材质、边框、图标、背景和动画。应用会在读取 CSS 前核对内置公钥签名、完整文件清单、大小和 SHA-256；通过后才写入 version 4 本地皮肤库与 IndexedDB。旧 `.workmode-skin.json`、未签名 ZIP 和旧本地皮肤库不会迁移。基础主题与签名皮肤先提供 palette，`frontend/src/themeContract.css` 再将其统一映射为 `--ui-*` 语义颜色；功能大厅、完整工作台与文献智库都从该层取得面板、控件、输入、浮层、文档、文字和状态颜色，并通过同一个 `SkinChrome` 渲染结构型顶栏。文献页面继续使用 `--lit-*` 作为领域别名，但不再拥有独立的深色默认值。项目笔记按钮还为图标与数量徽标分别保留固定槽位，窄头部中不再压缩图标。

GitHub Release 和桌面安装包只分发应用本体，不附带奖励皮肤。取得 `.workmode-skin` 后可在设置页一次选择多个文件导入并通过下拉菜单切换；若皮肤加载失败，应用会自动回到基础主题，若异常退出，下一次启动会停用未稳定的皮肤。紧急恢复快捷键 `Ctrl+Alt+Shift+R` 只清除皮肤选择并刷新，不会删除项目、会话或模型配置。维护机八套奖励皮肤现均已完成面向功能大厅、科研工作台和文献智库的三页面源码适配；八个签名包已重新生成并通过生产公钥验签。本地纠正版奶油泡芙 3.4.1、霜核机能台 3.3.1 和绿磷终端 4.2.1 已完成契约与验签，仍须在真实应用中重新导入、重启并逐页截图验收。不能把源码契约或验签通过等同于视觉完成。协议、签名边界和签包方式见 [官方签名皮肤](docs/CUSTOM-SKINS.md)。

奖励皮肤是独立于公开应用源码维护的签名内容包。它们的源码、设计样稿、字体、验签清单和生成后的 `.workmode-skin` 均保存在维护者本机被 Git 忽略的 `local-reference/reward-skin-library/`，不进入 GitHub、GitHub Actions、Release 或安装包。公开仓库只保留皮肤协议、导入器、运行时、安全恢复机制和离线签名工具。

维护机可运行 `scripts/build-skin-library.ps1` 从本地私有库重新签名；也可显式传入 `-SourceRoot` 与 `-PackageRoot`。保持同一皮肤 ID 并递增版本后，用户重新导入即可覆盖旧包，无需重装 Workmode。公开克隆不需要奖励皮肤源码即可构建和测试应用核心。

DeepSeek 官方预设使用 `https://api.deepseek.com`，可选择 `deepseek-v4-pro` 或 `deepseek-v4-flash`。预设只修改 Base URL 和模型名，不读取或覆盖 API Key；申请、充值、Key 管理和文档按钮通过受限的桌面权限交给系统浏览器，只允许打开 DeepSeek 官方域名。模型和价格可能变化，请以按钮打开的官方页面为准。

关闭主窗口会停止随应用启动的本地后端并退出。安装器、升级和卸载不会主动删除用户数据。

0.1.x 便携版用户可以在桌面版设置页选择「导入旧版便携包数据」。导入只复制旧目录的 `data/` 和 `config/.env`，不会修改旧目录，也不会覆盖已有的桌面版数据。

## 项目与会话

- 同一时间只有一个活动项目；切换项目时会加载该项目的会话、文件树和工作记忆。
- 科研工作台与文献智库共用同一套 session 标题和更新接口；工作台可双击会话名称或使用常驻的改名按钮，文献智库可在 session 选择器旁打开重命名窗口，修改后立即刷新列表而不改写历史。
- 新项目或切换项目时文件夹默认折叠；用户手动展开后，工具刷新文件树只保留仍然存在的展开目录。
- 会话列表和消息接口默认只加载最新 60 条，完整历史仍保存在本地 JSONL 中。
- 会话删除是软删除；原始 JSONL 历史保留在数据目录。
- 移除项目只取消 Workmode Public 中的注册关系，绝不会删除硬盘中的项目文件。
- 项目内新注册的子目录会显示在最近的已注册父项目下，同级项目按名称排列。

### 官方教程项目

左侧「创建教程项目」会让用户选择一个父目录，然后把安装包内的初始教程复制为独立项目并自动注册。教程包含真实开放获取 PDF、模拟实验数据、预计算事实报告、EPR 格式示例和项目级 `WORKMODE.md` 主持协议。

教程任务卡根据真实产品事件自动推进：打开 PDF、发送首个任务、完成网页检索、运行项目分析脚本、保存 Markdown 和查看上下文。任务按钮可以高亮操作区域或把建议指令填入输入框，但不会替用户自动发送消息。六项完成后显示正式项目入口并解锁「科研协作入门」成就。

只有通过应用创建、同时具有本地注册记录和有效 `WORKMODE_TUTORIAL.json` 标识的官方教程才显示「重置教程」。重置前会把完整项目目录、项目 Work Memory、结构化项目记忆和当前计划备份到 `%LOCALAPPDATA%\WorkmodePublic\data\work\tutorial-backups\`；随后恢复内置模板、软删除旧会话并新建空白会话。普通项目即使复制同名标识也不能调用该操作。

「重置教程」会清空六项教程任务进度，但不会抹掉已经解锁的历史成就；成就代表用户曾经完成过真实操作，不是教程文件状态。

## 文件与模型工具

模型直接加载必要工具，不依赖动态工具搜索：

- `project_list_dir`：列出一层目录；
- `project_glob`：按 pattern 查找文件；
- `project_grep`：用正则搜索 UTF-8 文本；
- `project_read`：按行号读取 UTF-8 文件；
- `project_write`：创建或整体重写文本文件；
- `project_edit`：对已有文本做精确字符串替换；
- `project_bash`、`project_python`：在当前项目根目录运行带超时和输出截断的命令或小型 Python 代码；
- `project_python_file`：使用安装包自带的 Python 直接运行项目内已有 `.py` 脚本，不要求用户另装 Python；
- `web_search`、`web_fetch`：通过 DuckDuckGo 免 Key HTML 搜索并行检索，并读取公开 HTTP(S) 文本资源；DuckDuckGo 无法连接时会返回明确的网络错误，不承诺无代理环境可用；
- `memory_write`、`memory_read`、`memory_list`：维护项目或全局工作记忆；
- `plan_my_steps`、`mark_step_done`：维护当前任务计划。

文件工具只接受当前项目中的相对路径，并拒绝越界路径、依赖/缓存目录和 `.env` 等敏感配置。命令工具不是系统级强沙箱，请只注册可信项目，并在重要修改后检查 diff 和测试结果。

## 固定上下文与工作记忆

项目根目录存在 `WORKMODE.md` 时，它会作为该项目专属 system prompt 在每一轮自动加载，并支持用独占一行的 `@相对路径` 展开项目内 UTF-8 文本。这个入口适合放项目纪律、教学主持协议等必须强制生效且应随项目一起分发的规则；它只影响当前项目，不会写入基础提示词、全局记忆或其它项目。

项目工作记忆、结构化工作记忆的索引和正文、当前计划都会固定进入 system prompt。若项目协议保存在文件中，可以在项目工作记忆里独占一行写：

```md
@docs/protocol.md
@INSTRUCTIONS.md
```

下一轮请求会从当前项目根目录读取这些 UTF-8 文本并注入上下文。绝对路径、越界路径、二进制文件和循环引用会显示为导入警告。

## 上下文窗口与压缩

完整会话历史包含用户消息、助手文字、工具调用和工具结果。模型侧不会无条件装入全部历史：

1. 先计算 system prompt、固定导入、工作记忆、计划和工具 schema；
2. 从配置的 Context Budget 中扣除这些固定部分；
3. 从最近历史向前选择能够完整装入的合法消息后缀，避免孤立工具结果；
4. 在顶部 token 条显示估算占用、已包含/丢弃的历史数量、固定文件和摘要状态。

点击「压缩上下文」会生成 `<CONTEXT_SUMMARY>` 标记。后续模型从最新标记继续，但标记之前的原始 JSONL 仍保留，压缩不是删除历史。

## 本地数据与安全边界

正式桌面版把用户状态保存在：

```text
%LOCALAPPDATA%\WorkmodePublic\
  config\.env        模型 API 与本地配置
  data\              项目注册、会话、记忆、计划和历史修复备份
  logs\runs\<run-id>\  最近 20 次桌面运行日志
  reports\           用户主动生成的错误报告 ZIP
```

官方教程重置备份位于 `data\work\tutorial-backups\<project-slug>\`，不会混入用户选择的教程工作目录。

默认后端只绑定 loopback。PDF/图片预览使用扩展名、大小与 magic bytes 白名单；网页工具拒绝 loopback、内网、链路本地地址、非常用端口和非 HTTP(S) 协议，并在每次重定向后重新验证目标。

Tauri 更新签名用于验证下载的更新内容，但当前安装器尚未配置 Windows Authenticode，因此首次下载仍可能触发 SmartScreen 提示。

快速反馈弹窗中的二维码随应用离线提供，不会自动上传诊断信息。每次桌面启动使用独立 `run-id` 记录桌面生命周期、前端未处理异常和后端标准输出/错误；Uvicorn 访问日志被关闭，避免请求 URL 中的本地路径或认证参数进入日志。用户点击「一键生成错误报告」后，Tauri 只读取本次运行目录，对 Token、API Key、密码和 Windows 本地路径再次脱敏，再把 `report.md`、`manifest.json` 与 `current-run.log` 写入 `reports\` 下的 ZIP，并在文件管理器中选中它。报告不会自动发送，生成后的 ZIP 也不会被应用自动删除。

## 开发与发行

仓库目录：

```text
backend/             FastAPI、上下文、会话、工具和文件存储
frontend/            React + Vite 多页面工作台（主工作台与文献智库）
desktop/             Tauri 2 桌面壳、后端生命周期和 NSIS 配置
tutorial-project/    随安装包分发的官方教程初始模板
literature-demo/     文献固定项目模板、领域说明与退役原型归档
scripts/             源码启动、验证和桌面发行脚本
docs/                架构、开发、发行与产品路线图
```

文献智库的活动前端位于 `frontend/src/literature/`，由主 Vite 工程构建为 `/literature/` 页面，并随 Tauri 的同一份 `frontend/dist` 进入正式安装包。根入口默认显示功能大厅，`?surface=workbench` 打开完整工作台；大厅只负责进入文献页面。文献页自身提供项目新建、切换、重命名和安全移除：新建时只填写名称，不再选择文件夹；Windows 有 D 盘时默认保存到 `D:\workmode\<项目名>`，否则使用当前用户的 `~/workmode`，可用 `WORKMODE_MANAGED_PROJECTS_DIR` 覆盖。同名目录自动追加序号。旧版任意路径文献项目继续原地注册和使用，首次访问只补齐缺失的固定结构，不移动 PDF、笔记、session 或其它数据。移除项目只取消应用注册，实体目录仍由用户在文件管理器中手动删除。完整工作台不再保留跨模块快捷按钮；文献智库也采用相同的 48px 左侧活动栏，两者都只提供返回大厅和打开设置。文献设置通过 `?surface=workbench&panel=settings&return=literature` 复用全局设置，关闭时原路返回。文献顶部只显示模块品牌与当前项目，不再重复显示后端连接或特化模式徽章；连接故障仍在对话区给出可操作错误。文献详情、项目笔记和项目记忆统一使用脱离主网格的居中模态窗口；原始 PDF 通过共享 `PdfViewer` 从正式 `/fs/media` 白名单端点获取为本机 Blob，再交给浏览器原生 PDF 界面显示，与工作台文件阅读器保持同一实现，并避免第三方下载接管程序拦截原始 HTTP 导航。三个界面共享桌面动态后端地址、本地认证 token、主题/签名皮肤基础和正式项目注册。文献对话直接消费正式 Workmode SSE：用户消息先显示，正文增量与工具开始/结果按到达顺序更新，结束后再从 JSONL 回读校准。`literature-demo/` 只保留固定项目模板、领域说明与退役原型归档。旧 `/api/literature-demo`、独立 store、独立 session、独立压缩器和“提案 → 确认”写入链已经移出运行路径；历史代码位于 `literature-demo/archive/standalone-backend-2026-07-13/`，不会进入 FastAPI 或 Release。

正式后端把文献库识别为一种固定结构的 Workmode 项目。项目根目录的 `literature-project.json` 声明 `project_type=literature-library`；`catalog.json`、`tags.json`、`papers/`、`notes/`、`exports/` 和 `processed-index.md` 是权威数据。`WORKMODE.md` 通过 `@LITERATURE_PROJECT.md` 固定注入项目纪律。对话、多 session、JSONL 工具事件、最近 60 条前端回放、token 窗口、压缩、停止生成和项目 memory 全部复用正式 Workmode 内核。

文献项目只向模型暴露 15 个 `literature_*` 领域工具：文献检索、标签注册表读取、正文/事实读取、导入、MinerU/事实抽取、记录更新、跨文献段、归档、文献删除/恢复、笔记检索/读取/写入/删除/导出。`literature_delete` 把目录记录、PDF 和解析产物作为一个事务移入 `papers/.trash/<时间>--<paper-id>/`，`literature_restore` 按清单原路恢复且拒绝覆盖已有路径；前端详情按钮与文献回收站使用同一领域服务。删除只清理当前活动投影和选择，不改写历史 JSONL 中曾经导入或选择该文献的事实。笔记删除同样进入项目内 `notes/.trash/`，固定的 `notes/README.md` 不可删除；导出物保持现有手动管理方式。`literature_read` 可接受单个 `paper_id` 或一组 `paper_ids`；普通内容问题优先读取既有 MinerU Markdown，没有时直接读取 PDF 文本层，不会因此启动增强流水线。`literature_tag_list` 会返回规范标签 ID、名称、别名、分类、确认状态和引用论文数；记录更新工具明确要求写标签前先查注册表并优先复用，避免盲打产生重复标签。文献模式不加载 Shell、Python、通用文件、Web、通用 memory 或 plan 工具。领域写工具调用后直接执行，不需要 proposal、二次批准、`confirmed` 参数或“确认”关键词；后端仍校验项目身份、固定目录、paper ID、JSON schema、受控路径和原子写入。前端勾选的文献/笔记作为 `active_context` 写入当轮用户消息元数据，只用于优先上下文，不是权限条件；论文选择发生变化时还会在该 session 中留下永久 system 事件，回看历史可以确认当时讨论的材料。顶部项目名与上下文百分比直接读取正式 Workmode 项目和 context API，不再保留静态 Demo 值或独立任务轮询。工具轮次之间显示的“继续生成”只属于流式临时状态；收到本轮 `done` 或最终回读完成后立即清空，不会残留到下一次发送。

PDF 拖入后先显示文件清单，只有用户点击「确认入库」才通过正式项目投影接口流式写入 `papers/unprocessed/pdf/`，并以 SHA-256 去重。成功记录只进入项目文献库，不会自动加入当前选择，也不会调用模型或处理流水线；后端在当前 session 中追加一条仅包含真实文件名和 paper ID 元数据的确认入库 system 事件。该事件在下一次用户发送消息前进入模型上下文并显示在正式时间线中，之后永久保留。文献选择变化同样在对应 session 中保存 system 事件；同一选择连续对话不会重复写入。`literature_read(part="full_text")` 优先读取 MinerU `full.md`；若 MinerU 未配置、失败或结果不存在，则直接读取原 PDF 的本地文本层并明确标记来源。MinerU 不是必需项，只有用户明确要求增强解析、结构化事实或归档，或普通文本层不足以回答时才使用；配置后会明显提高多栏正文、表格、公式和版面顺序的识别准确性，扫描件或纯图片 PDF 仍需 MinerU/OCR。`literature_process` 在同一高级工具调用中运行 MinerU、首页元数据抽取、标准命名和六段客观事实报告；传入 `paper_ids` 时最多并发处理 3 篇并逐篇返回结果。元数据请求使用 JSON object 响应约束并校验 `Cite This`/`layout.json` 证据是否真实存在；解析失败只修复一次，仍失败则把问题写入论文记录并继续生成事实报告。设置页可保存 MinerU Token、`pipeline`/`vlm`、语言和 60–1800 秒超时，默认 180 秒。事实报告保存为 `papers/unprocessed/extracted/<paper-id>/objective-facts.md`；归档会先检查 PDF 与解析目录的全部目标位置，再同步移动产物并重建 `processed-index.md`，目标冲突不会留下只移动了一半的 PDF。

源码一键启动器会验证 `backend/.venv` 中的 Python 是否仍可执行；若系统 Python 被移动或卸载导致虚拟环境失效，会在安全校验路径后自动重建。安装依赖时的 pip 输出只写到终端，不再混入后续 `Start-Process` 的可执行文件路径；启动器也会规范化 Windows 环境中的 `Path`/`PATH` 重复项。

开发机上的 `.run/` 只保存 PID、临时发行配置、可重建 smoke 目录和项目自带开发工具。奖励皮肤维护库及其他不能提交的本地参考材料放在 `local-reference/`，其内容默认被 Git 忽略；公开应用构建与测试不得依赖这些本地文件。

## 当前边界

- 正式安装包目前只发布 Windows x64；
- `project_bash`、`project_python` 和 `project_python_file` 有项目 cwd、超时、输出截断和取消能力；shell 另有破坏性命令黑名单，但这些工具都不是 OS 级隔离；

- 尚未配置 Windows Authenticode。
