# 文章处理使用指引

文章处理是一个不创建项目、不创建对话的轻量文字工具。你粘贴一段或一篇文章，选择“文字润色”或“查找漏洞”，处理成功后就能直接复制结果，并在左侧历史中随时重新打开。

## 第一次使用：先配置模型

文章处理调用你自己选择的 OpenAI-compatible 模型服务。没有配置模型时，页面顶部会显示“尚未配置模型”。

1. 打开 Workmode Public，在功能大厅点击“文章处理”。
2. 点击右上角“模型设置”。
3. 在“模型 API”卡片填写服务商提供的 **Base URL**、**模型名**和 **API Key**。
4. 点击连接测试。看到成功提示后再保存。
5. 返回文章处理页；顶部显示模型名称，就表示可以开始使用。

Base URL、模型名和 API Key 必须来自同一家服务商，不能把不同平台的三项配置混在一起。连接失败时先检查：

- API Key 是否复制完整，前后有没有多余空格；
- Base URL 是否是服务商给出的 OpenAI-compatible 接口地址；
- 模型名是否与该账号实际开通的模型完全一致；
- 账号是否还有余额、额度或调用权限；
- 服务商是否对网络、地域或并发有限制。

Workmode Public 不替你注册模型账号，也不会在页面中回显已经保存的完整 API Key。不同服务商的价格、隐私和数据保留规则不同，提交前请查看你所选服务商的最新说明。

## 文字润色

适合处理摘要、论文段落、报告、回复意见或其它需要改善表达的文字。

1. 选择“文字润色”。
2. 把原文粘贴到左侧“文字输入”。
3. 点击“开始处理”。
4. 等待右侧出现结果，快速核对事实、数字、引文和结论强度。
5. 点击“复制结果”，粘贴到 Word 或其它编辑器。

润色规则要求模型改善清晰度、连贯性和学术表达，同时保留原文的事实、数字、引用、限定条件和结论强度。如果原文存在事实错误，润色不会自动证明或纠正它；重要内容必须由作者核对。

## 查找漏洞

适合检查较长文章内部的证据链、论证逻辑和前后一致性。

1. 选择“查找漏洞”。
2. 尽量粘贴完整段落或完整文章。只给一句话时，模型看不到上下文，能够发现的问题会更少。
3. 点击“开始处理”。
4. 在右侧查看报告，按严重程度回到原文逐项核对。

报告重点检查：

- 结论是否缺少前提或证据支撑；
- 相关性是否被写成因果关系；
- 概念、对象、范围、时间或样本是否中途发生变化；
- 数字、术语、缩写和结论是否前后矛盾；
- 绝对化表述是否超过现有证据强度；
- 段落之间是否存在逻辑跳跃、遗漏的反例或未解释的限制。

长文会先分段分析，再合并成全篇报告。这个功能**不会联网检索**，也不会替你验证外部事实、原始实验数据或参考文献是否真实存在。它发现的是输入文本内部的风险线索，不是同行评审或事实核查的替代品。

## 上标和下标

模型被要求优先输出可直接复制的 Unicode 上下标和下标，例如：

```text
H₂O
CO₂
10⁻³
R²
xⁿ
```

后端也会把含义明确的常见 HTML/LaTeX 写法转换为 Unicode，例如 `<sub>2</sub>`、`^{-3}` 和 `R^{2}`。Unicode 本身只覆盖一部分字符；复杂公式、分式、根号和长表达式仍应使用 Word 公式或 LaTeX 编辑器，不会被强行转换。

## 处理历史

只有模型成功返回结果后，Workmode Public 才会创建历史记录。每条记录保存：

- 原始输入全文；
- 当时选择的功能；
- 输出全文；
- 使用的模型、处理时间和字数。

点击左侧记录可以恢复当时的输入和输出。修改输入后再次点击“开始处理”会创建一条新记录，不会覆盖旧结果。点击“新任务”会清空输入、输出和当前历史选中状态，但保留“文字润色”或“查找漏洞”的当前选择，也不会删除任何历史；模型正在处理时该按钮会暂时禁用。删除会把记录移入“已删除记录”；点击“恢复”可以找回。当前版本没有自动清空历史的后台任务。

正式桌面版把记录放在应用数据目录的 `article-processing/history/`，已删除记录位于 `article-processing/.trash/`。源码版的具体根目录取决于 `WORKMODE_PUBLIC_DATA_DIR` 或当前系统默认配置；页面的运行状态会显示实际使用的历史根目录。历史列表只读取时间、模式、字数和短预览，点击某条时才加载输入与输出全文，因此长文记录增多后不会在每次刷新时全部传到前端。

## 隐私、费用与安全边界

- 输入文字会发送给你在设置中配置的模型服务，模型服务可能计费，也可能按其条款保存请求。
- 输入与输出成功后还会完整保存在本机历史中。共用电脑上的其他本机账户权限、备份软件或恶意软件可能访问这些文件。
- 不要在没有授权的情况下粘贴患者信息、未公开数据、商业秘密、个人身份信息或受保密协议约束的内容。
- Workmode Public 不会因为打开文章处理页就自动请求模型；只有点击“开始处理”才会发送文字。
- 处理失败不会创建一条看似成功的历史，但模型服务仍可能已经收到请求或计入调用费用。
- 删除是可恢复移动，不是安全擦除。如果内容必须彻底销毁，还要按你所在组织的数据治理要求处理回收记录、系统备份和模型服务端数据。

## 常见问题

### 点击“开始处理”后提示尚未配置模型

打开右上角“模型设置”，补齐 Base URL、模型名和 API Key，连接测试成功后保存，再返回重试。

### 处理很慢或超时

长文会产生更多模型调用。可以先分章节处理，或检查模型服务商的并发、上下文长度、余额和服务状态。不要通过修改系统代理、DNS 或其它网络环境来绕过应用错误；先确认模型配置和服务商自身是否可用。

### 润色结果改变了原意

不要直接覆盖原稿。打开历史记录对照原文，重点检查数字、否定词、条件范围、因果关系和结论强度，然后修改输入或换用更合适的模型重新处理。

### 漏洞报告说某条引用有问题，是真的吗

它只能指出引用和论证在当前文章中的连接可能不足，不能证明引用真伪。请打开原始文献或权威数据源人工核验。

### 为什么某个公式没有转换成上下标

Unicode 上下标/下标字符集不完整。简单数字、正负号和常见字母可以直接转换，复杂公式应保留为 Word 公式或 LaTeX，避免转换后含义错误。
