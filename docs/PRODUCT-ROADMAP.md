# 产品路线图

本文记录当前已交付基线和仍需明确设计的产品方向。它不是发布日期承诺；只有已经实现并验证的能力才会进入“已交付”。

## 已交付（v0.8.10）

- 文献智库新增 EndNote 文献库迁移：支持 `.enl` + 同名 `.Data` 与 `.enlx`，自动查找可扫描本机固定盘与可移动盘；导入有价值的元数据、手工分组、真实彩色标签、主 PDF 与 SI，不调用模型或 MinerU，也不修改原库；
- 删除前端硬编码的五个假标签组，标签组改为项目中的真实数据实体；新增发表日期、文献分组筛选、每篇文献独立 SI 文件夹及“打开 SI 文件夹”；
- 导入后可按 DOI、主 PDF SHA-256、规范化标题+年份+第一作者扫描重复候选；当前只报告，不自动合并；
- 文献前端、后端字段与 AI 工具统一到同一份元数据、文献分组、彩色标签和附件语义；缺失作者、年份或期刊时明确标记元数据不完整；
- 文献列表和详情页改为紧凑阅读布局，统一搜索、筛选与导入控件，移除原始文件名等内部信息；顶部 HUD 统一承载项目入口，并为全部按钮提供悬浮提示；
- 文献新手引导先检查模型 API，未配置时高亮设置入口；配置后直接在真实界面依次高亮项目、导入、筛选、文献列表和对话输入区，不再弹出多页说明窗口。

## 已交付（v0.8.7）

- 功能大厅扩展为四张统一尺寸的 2×2 入口卡，并新增无 session 的“文章处理”入口；
- 文章处理提供“文字润色”和“查找漏洞”两个模式，前者要求保留事实、数字、引用和结论强度，后者核查输入内部的证据链、逻辑跳跃、概念漂移与表述一致性；
- 长文漏洞核查采用分块局部分析后再全篇合并，不把内部文本检查伪装成联网事实或参考文献核验；
- 明确的简单上下标尽量转换为可直接复制进 Word 的 Unicode 字符，例如 `H₂O`、`10⁻³` 与 `R²`；
- 成功处理记录以原始输入、模式、输出、模型、时间和字数完整保存在本机，历史列表使用轻量预览，详情按需读取，并支持可恢复删除；
- 润色与核查规则由版本化、纯文本的内置 Skill 文件加载，不包含 Bash、网络或项目文件权限。

## 已交付（v0.8.6）

- 新增无 session 的“会议录音转文字”入口：一次上传多个录音，使用 DashScope Fun-ASR 区分说话人，并集中查看带时间戳分段、纯文本和 Markdown；
- 转写工作区固定为 `tools/input/output`，列表只扫描 `output/<任务 ID>/meta.json`；同一根目录由通用工作台增加的其它文件不会进入转写状态；
- 保存 Fun-ASR 远端任务 ID 并在重启后恢复轮询，提供失败重试、标题修改、TXT/Markdown/JSON 下载、输入输出成对软删除和回收站恢复；
- DashScope API Key 进入共享设置页，只在本机 `.env` 保存且不回显，不写入录音、结果或日志；
- 转写完成后可手动使用共享 OpenAI-compatible 模型生成忠实润色稿或结构化会议总结；长文本分段处理，原始 ASR 永不覆盖，派生 Markdown 可查看、下载、重新生成和清除，并通过源指纹防止与重新转写结果错配；

- 皮肤从有限积木系统升级为“内容槽位 + 布局 CSS + 视觉 CSS”：同一批真实项目、文件、会话、消息、工具和设置可以由不同皮肤重新排布和重绘；
- 用户入口只接受官方 Ed25519 签名的 `.workmode-skin`，验签覆盖全部文件路径、大小与 SHA-256；旧 JSON、未签名 ZIP、未知公钥和被修改的包全部拒绝；
- 本地皮肤库升级为 version 4，旧 version 1/2/3 与旧 IndexedDB 不迁移；切换和移除会清理注入样式、Blob URL 与 FontFace；
- 增加启动 boot guard、运行时自动回退和 `Ctrl+Alt+Shift+R` 紧急恢复，皮肤故障不触碰项目、会话、JSONL 或模型配置；
- 新增离线官方签包工具；奖励皮肤源码、样稿、字体、验签清单和签名包统一迁入维护机被 Git 忽略的 `local-reference/reward-skin-library/` 并手动发放，公开仓库与 Release 只发布应用核心、协议和签名工具；
- 设置页提供低门槛本地错误报告：按本次运行生成经过密钥与路径脱敏的 ZIP，并在 Windows 文件管理器中直接选中；应用不上传、不代发，用户自行拖入“研天雪”公众号私信、邮件或其它渠道。

### 本地科研工作台

- Windows x64 安装器，目标电脑不需要 Node.js、Python 或 Rust；
- 多项目注册和切换、真实目录父子层级、活动项目持久化；文件夹默认折叠并按科研文件类型显示图标；
- 会话重命名、会话软删除、项目安全移除；
- UTF-8 文本、Markdown、PDF 和图片浏览，Markdown 编辑；AI 回复和文件预览支持 GFM 表格与宽表横向滚动；
- 可调整宽度的文件面板和长文档独立滚动。
- 安装包内置科研协作教程，可在应用内创建；官方教程支持有备份、保留历史的安全重置。

### 模型工作能力

- 项目读、写、精确编辑、目录、glob、grep、shell、内联 Python 和内置 Python 脚本运行工具；
- 工作记忆和计划工具；
- 并行 `web_search` 和 `web_fetch`，带公开网络边界；
- 所有必要工具固定加载，不依赖动态工具搜索；
- 工具循环没有固定轮数上限，可由用户停止。

### 上下文与历史

- 项目根 `WORKMODE.md` 专属提示词及 `@相对路径` 固定导入；
- 项目工作记忆中的 `@相对路径` 固定导入；
- 结构化工作记忆索引和正文固定注入；
- token 预算动态历史窗口和前端占用指示；
- 手动上下文压缩，保留完整 JSONL；
- 最近 60 条前端分页加载；
- “文字 → 工具 → 文字”持久化顺序、工具卡原位更新、停止后的已取消状态；
- 启动时备份并修复旧版悬空工具事件。

### 桌面产品化

- 单实例、托盘、动态本地端口和后端进程树生命周期；
- GitHub Releases 检查更新、下载进度、Tauri 签名验证和应用内安装；

- 0.1.x 便携数据非破坏性导入。

## 下一阶段：文献研究流水线

当前网页工具适合基础检索和原文核对，但还不是专业文献管理系统。候选范围：

1. 从研究问题生成可检查的多组 query；
2. 优先学术/一手来源，记录标题、作者、期刊、年份、DOI、URL 和检索时间；
3. 按 DOI、规范化标题和 canonical URL 去重；
4. 区分来源原文、模型推断和无法验证的结论；
5. 让用户选择哪些结果进入项目上下文；
6. 导出可追踪引用和项目内研究记录；
7. 为一次调研设置搜索次数、token、时间和取消预算。

需要先确定搜索供应商、凭据、限流、PDF/全文权限、引用格式和失败降级语义。

## 下一阶段候选：受限子 agent

暂不加入自主子 agent，直到以下边界有可测试的设计：

- 子任务允许访问哪些工具和项目路径；
- 最大并发、token、时间和网络预算；
- 只读、建议修改和直接编辑角色的区别；
- 同时修改同一文件时的隔离、冲突检测和合并；
- 继承哪些上下文，返回何种结构化结果；
- 主对话如何显示进度、来源、失败和取消；
- 委派由用户显式发起、用户确认，还是允许模型自主决定。

最小可行候选是“有界、只读、可取消的文献检索团队”。代码编辑 agent 应在文件冲突和批准语义成熟之后再考虑。

## 明确非目标

- 不恢复私人伴侣、生活记忆或关系人格模块；
- 不把用户项目、会话或 API Key 默认上传到云端；
- 不把 shell/Python 描述为安全容器；
- 不在来源不可验证时伪造论文、DOI、实验结果或引用。
