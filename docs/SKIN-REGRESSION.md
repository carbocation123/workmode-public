# 官方皮肤回归基线

本文固定 0.7.0 官方签名皮肤的自动与人工检查口径。签名证明来源和完整性，不证明布局一定正确；每次修改语义槽位、导入器、资源运行时、恢复机制或正式皮肤 CSS，都必须重新验证。

## 自动契约

前端测试必须覆盖：

- 使用临时 Ed25519 密钥生成的合法包可以导入；缺签名、未知 keyId、文件篡改、额外文件、路径越界和旧 JSON 会被拒绝；
- 临时 Ed25519 测试包能通过导入器完整验签；本机 `skin-library/packages/` 中的奖励包在手动发放前还要用内置生产公钥逐个验签；
- 验签在 manifest 与 CSS 解析之前发生；
- `layout.css`、`visual.css`、字体和图片只在当前活动皮肤加载，切换后撤销 `<style>`、Blob URL 与 FontFace；
- version 4 本地库只接受带签名收据的皮肤，旧 version 1/2/3 和旧 IndexedDB 被清理；
- React 持续提供项目、文件、会话、聊天、消息、工具、输入区、查看器、设置和状态栏等稳定 `data-skin-slot`；
- boot guard、运行时失败回退和紧急恢复不会修改项目、会话、JSONL、工作记忆或模型配置。

后端/发行契约必须确认 Release 构建脚本不读取 `skin-library/`、不包含 `.workmode-skin`，GitHub Actions 只上传 Release 根目录中的应用产物，也不会复制 `.release-secrets` 或皮肤私钥。

## 人工检查表

| 区域 | 必须验证 |
| --- | --- |
| 启动 | 启用皮肤后重启正常；模拟加载失败后自动回到基础主题；`Ctrl+Alt+Shift+R` 可恢复 |
| 项目与文件 | 项目/设置导航、折叠目录、长文件名、Markdown/PDF/图片/代码/数据/压缩/文本/媒体图标、会话图标以及打开/刷新/移除/重命名按钮均符合当前皮肤语言；不能退回系统 Emoji，右键/删除入口可见且可操作 |
| 图标槽位 | 自定义图标必须占用组件预留槽位，不能覆盖文件名、项目名或会话名；替换型伪元素必须隐藏底层回退字形，不能让一个按钮同时出现两套标识 |
| 行内代码 | 亮色消息纸面与文件纸面中的路径、命令和行内代码必须保持足够前景/背景对比；代码块内部不能重复套用行内标签边框 |
| 会话 | 新建、切换、重命名、软删除均可达；选中态不只依赖颜色 |
| 消息 | 用户/助手角色可辨，Markdown 列表与表格不溢出，停止后的文字—工具—文字顺序不变 |
| 工具 | 运行、完成、失败、取消状态清楚；详情折叠可操作；卡片不过度占高 |
| 输入区 | 始终位于聊天工作区底部，`Ctrl+Enter` 可发送，停止按钮和错误提示不被遮挡 |
| 文件查看器 | 长 Markdown 独立滚动，PDF 高度填满面板，图片可见，拖动分隔条后仍能关闭 |
| 设置 | 宽屏与窄屏均能进入所有设置分区；皮肤选择器、移除和恢复入口不被皮肤隐藏 |
| 上下文与状态 | token 百分比、历史数量、模型和生成状态来自真实数据，不出现假遥测 |
| 响应式 | 1920×1080、1366×768、最小支持窗口与 125%/150% Windows 缩放均能完成核心操作 |
| 可访问性 | 键盘焦点可见；降低动效生效；正文对比度可读；背景和装饰不截获点击 |

## 代表皮肤

- 绿磷终端：高保真基线，重点检查像素字体、CRT 扫描线、文件正文内边距与宽度拖动、token 标签单行容纳、头部上下文/历史/操作按钮间距、Markdown 绿色语义、AI 左/用户右共轨角色标签、终端工具状态槽和小窗口降级。
- 奶油泡芙：亮色厚材质基线，重点检查矢量兔兔、低对比文字、按钮深度和设置页可达。
- Neon Ice：透明 HUD 基线，重点检查 2px 玻璃边线、无切角结构、文件查看器拖动和窄屏降级。
- 霜核机能台：明暗混合材质基线，重点检查银白侧栏、深色消息区、能量核心和亮色文件纸面之间的对比度。
- 午夜控制台：复古设备基线，重点检查真实状态字段、电台装饰、角色标签、寻呼机工具卡和保密卷宗不互相遮挡。
- 紫晶星象塔：暗色装饰基线，重点检查星空、星盘、角色铭牌和装饰层不覆盖正文与安全按钮。
- 酒保行动·像素夜班：奖励皮肤基线，重点检查像素字体加载、完整矩形对话框、真实工具类型标签、整条上下文仪表带、项目/设置/文件类型/会话像素图标、打开/循环刷新/废纸篓移除/重命名按钮、角色名牌、城市剪影和最小窗口可操作性；图标必须保持 2px 像素网格、占用独立槽位且不能退回系统 Emoji。与其余签名包一样只手动发放，不得进入公开 Release。

## 截图基线

每套准备手动发放的奖励皮肤至少保存四张真实应用截图：空会话、包含 Markdown 的对话、包含完成/失败/取消工具卡的对话、打开文件查看器。路径为：

```text
design/skin-lab/regression/<version>/<skin-id>/<scene>.png
```

截图不能由静态样稿代替。尚未生成截图时必须记为待办，不能声称视觉回归已经完成。
